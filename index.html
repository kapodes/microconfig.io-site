Micro©onfig is a powerful tool for microservice configuration management

Easy to reuse common part of configuration

Copy-paste free configs

Powerful expression language support

For dynamic properties configuration

Designed to be used with any programming languages

Collection of best practices for configuration management

Features

  • Includes
  • Placeholders
  • Expression language
  • Env specific config
  • Templates support
You can extract and reuse common configuration:
#include service-discovery-client, logger-configuration
You can use placeholders to another component's properties:
order-service.url=http://${order-service@host}:${order-service@port}
You can use powerful EL for dynamic properties:
mainClassNameWithoutPackage=#{'${this@java.main}'.substring('${this@java.main}'.lastIndexOf('.') + 1)}
You can override default configuration by env-specific:

application.yaml - file for common configuration
application.dev.yaml - overrides for dev env
application.prod.uat.yaml - overrides for prod and uat envs
You can resolve placeholders inside arbitrary configs files.
